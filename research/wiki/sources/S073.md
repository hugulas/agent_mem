# OpenClaw docs: Honcho memory

> Official docs | 2026-05-06

## 1. Core Claim
- Honcho adds AI-native cross-session memory to OpenClaw by persisting conversations to a dedicated service, automatically building user and agent models over time through semantic search and multi-agent awareness.

## 2. Method / Mechanism
- **Cross-session persistence**: Conversations persisted after every turn; context carries across session resets, compaction, and channel switches.
- **User modeling**: Maintains profiles for each user (preferences, facts, communication style) and for the agent (personality, learned behaviors).
- **Semantic search**: Search over observations from past conversations, not just current session.
- **Multi-agent awareness**: Parent agents track spawned sub-agents; parents added as observers in child sessions.
- **Five registered tools**:
  - Data retrieval (fast): `honcho_context`, `honcho_search_conclusions`, `honcho_search_messages`, `honcho_session`
  - Q&A (LLM-powered): `honcho_ask` (depth=`quick` for facts, `thorough` for synthesis)
- **Integration phase**: Injected during `before_prompt_build`, ensuring accurate turn boundaries and relevant recall.
- **Deployment**: Self-hosted (local server) or managed API at `api.honcho.dev`.
- **Migration**: Non-destructive upload of existing workspace memory files (`USER.md`, `MEMORY.md`, `IDENTITY.md`, etc.).

## 3. Key Evidence
- Official OpenClaw documentation — authoritative integration reference.
- Comparison table: Honcho vs. builtin/QMD memory across storage, cross-session, user modeling, search, multi-agent, and dependencies.
- CLI commands documented: `openclaw honcho setup`, `openclaw honcho status`, `openclaw honcho ask`, `openclaw honcho search`.

## 4. Limitations & Gaps
- Plugin architecture adds a dependency layer; self-hosted option requires additional infrastructure.
- No quantitative benchmark data (latency, recall accuracy) provided in the docs.
- Migration uploads local files to remote service — privacy implications not deeply addressed here (see S076).
- Semantic search quality depends on embedding model choice, which is not specified.
