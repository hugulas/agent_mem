# MIT AI Agent Index: UI-TARS-desktop

> Academic index | 2025-03

## 1. Core Claim
- The 2025 AI Agent Index provides the most rigorous third-party documentation of 30 deployed agentic AI systems, finding that developer transparency is poor—most safety-related fields (135/240) have no public information, and nearly all indexed agents rely on just three foundation model families (GPT, Claude, Gemini).

## 2. Method / Mechanism
- Systematic indexing framework with 45 information fields across 6 categories: product overview, company accountability, technical capabilities, autonomy & control, ecosystem interaction, and safety & evaluation.
- Three agent categories: chat applications with agentic tools (12), browser-based agents (5), and enterprise workflow agents (13).
- Annotations based solely on public documentation, demos, and correspondence—no experimental testing.

## 3. Key Evidence
- **30 agents indexed** across chat, browser, and enterprise categories.
- **135 of 240 safety-related fields** had no publicly available information.
- Nearly all agents rely on **GPT, Claude, or Gemini** families.
- Only **4 agents** provide agent-specific safety evaluations.
- UI-TARS Desktop specifically categorized as browser/system-based agent with CLI + web UI, Apache 2.0 license.
- For UI-TARS Desktop: long-term memory retains history across interactions, but **no evidence of episodic memory across different conversations**.

## 4. Limitations & Gaps
- No experimental testing or benchmarking of agents—the analysis is purely documentary.
- Cutoff date of December 31, 2025 means very recent developments (e.g., UI-TARS-2, early 2026 releases) may be missing.
- UI-TARS memory assessment is limited to publicly visible features; internal ByteDance architecture details not available.
- Chinese ecosystem expertise was consulted but may still miss region-specific nuances.

## 5. Relevance to Our Review
- Feeds into **"ByteDance agents"** and **"UI-TARS memory"** sections.
- Provides independent third-party validation (or lack thereof) of UI-TARS Desktop's memory claims.
- Useful for ecosystem-wide context on how agent memory transparency compares across vendors.
