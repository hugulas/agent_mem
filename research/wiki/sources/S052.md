# Microsoft Learn: Agent memory governance

> Official docs | 2025-12

## 1. Core Claim
- Microsoft prescribes an enterprise governance framework for agent memory that emphasizes aligning persistence models with agent function, externalizing state storage, enforcing strict isolation boundaries, and automating retention and disposal.

## 2. Method / Mechanism
- **Align persistence with function**: Memory retention should match the operational purpose of the agent; don't store more than necessary.
- **Externalize state storage**: Agent memory should be stored outside the agent runtime in managed services, not embedded in prompts or local files.
- **Strict isolation boundaries**: Prevent cross-tenant and cross-user memory leakage through access controls and sandboxing.
- **Automate retention and disposal**: Define TTLs, automatic purging, and compliance-driven deletion policies.

## 3. Key Evidence
- Official Microsoft Learn documentation provides the governance framework.
- Four pillars structure the guidance: alignment, externalization, isolation, automation.

## 4. Limitations & Gaps
- Source is a 404 page; details come from reading-log summary only.
- No technical implementation guidance—framework is policy-oriented rather than engineering-specific.
- No benchmarks or case studies showing effectiveness of the prescribed governance model.
- Doesn't specify which Azure services should implement these controls (Cosmos DB, Azure AI Search, etc.).
