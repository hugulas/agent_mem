# antigravity.codes: Karpathy's 6-Step Workflow

> Workflow breakdown | 2026-04

## 1. Core Claim
- Karpathy's knowledge-base workflow can be decomposed into six reproducible steps: data ingest → LLM compilation → scale → querying → multi-format output → health checks, with the LLM functioning as a compiler in a structured pipeline.

## 2. Method / Mechanism
- **Six-step workflow**:
  1. **Data Ingest**: Collect raw sources into a structured input directory.
  2. **LLM Compilation**: Use LLM to transform raw inputs into curated markdown wiki articles.
  3. **Scale**: Grow the wiki organically as new sources are added.
  4. **Querying**: Ask questions against the compiled wiki rather than raw sources.
  5. **Multi-format Output**: Generate presentations (Marp), visualizations (matplotlib), or other formats from the wiki.
  6. **Health Checks**: Automated linting/validation ensures consistency and flags issues.
- **Compiler Analogy**: The LLM is not a retriever but a compiler that transforms unstructured inputs into structured knowledge.

## 3. Key Evidence
- Step-by-step decomposition makes the workflow reproducible.
- Explicit health-check stage addresses quality maintenance.
