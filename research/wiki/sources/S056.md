# AWS: Build persistent memory for agentic AI

> Official blog | 2025-11

## 1. Core Claim
- AWS recommends a native cloud memory stack for agentic AI combining Mem0 for memory extraction/management, Amazon ElastiCache for Valkey as the vector store, and Amazon Neptune Analytics for graph memory backends.

## 2. Method / Mechanism
- **Mem0**: Handles memory extraction, consolidation, and retrieval operations.
- **Amazon ElastiCache for Valkey**: Serves as the high-performance vector store for semantic retrieval.
- **Amazon Neptune Analytics**: Provides graph memory backend for relationship-aware memory queries.
- **AWS-native deployment**: Fully managed services with automatic scaling, backup, and security.

## 3. Key Evidence
- Reference architecture published by AWS Machine Learning blog.
- Combines three AWS services into a unified memory pipeline.
- Graph memory backend (Neptune) enables relationship traversal beyond flat vector search.

## 4. Limitations & Gaps
- Source is a 404 page; details come from reading-log summary only.
- No performance benchmarks (latency, throughput, accuracy) for this specific stack.
- No cost analysis for running all three services at scale.
- Mem0 is a third-party component; dependency on external open-source project introduces maintenance risk.
- No discussion of multi-tenant isolation or data residency compliance.
