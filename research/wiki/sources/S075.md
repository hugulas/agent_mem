# Plastic Labs blog: Beyond User-Assistant Paradigm

> Official blog | 2025-08-18

## 1. Core Claim
- Honcho's "Peer" architecture moves beyond the simple user-assistant dyad by introducing an App→User→Session→Message hierarchy with peer profiles for multi-agent scenarios and directional observation modes.

## 2. Method / Mechanism
- **Hierarchy**: App → User → Session → Message — four-level nesting for context isolation.
- **Peer Profiles**: Each agent instance talking to the same user maintains a separate peer profile, preventing cross-contamination.
- **Directional Observation Modes**:
  - **Unified observation**: All peers share observations (default for single-agent scenarios).
  - **Directional observation**: Peers observe only specific directions/flows, enabling fine-grained privacy and context isolation in multi-agent setups.

## 3. Key Evidence
- Architecture rationale from Plastic Labs (Honcho's creators).
- Explicit hierarchy and observation-mode design.

## 4. Limitations & Gaps
- **Source inaccessible**: Web archive contains mostly empty HTML; content from reading log only.
