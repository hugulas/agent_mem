# Graph-Native Cognitive Memory for AI Agents

> Academic paper | 2026-02-04

## 1. Core Claim
- The structural primitives required for cognitive memory (immutable revisions, mutable tag pointers, typed dependency edges, URI-based addressing) are identical to those needed for managing agent-produced outputs in multi-agent workflows.
- A single graph-native architecture grounded in formal belief revision semantics can simultaneously serve as cognitive memory and operational infrastructure for multi-agent work.

## 2. Method / Mechanism
- **Kumiho architecture**: Graph-native cognitive memory with formal AGM belief revision semantics.
- **Dual-store model**: Redis working memory + Neo4j long-term graph with hybrid retrieval (full-text + vector).
- **Asynchronous consolidation pipeline**: Extends sleep-time compute with safety guards (published-item protection, circuit breakers, dry-run validation, auditable cursor-based resumption).
- **Unified graph**: Same graph stores memories and manages work products; downstream agents locate inputs via URI resolution and track revisions via tag pointers.

## 3. Key Evidence
- **LoCoMo benchmark**: 0.447 four-category F1 (n=1,540)—highest reported score across retrieval categories. Overall F1 0.565 (n=1,986) including adversarial scoring.
- **97.5% adversarial refusal accuracy** (n=446), attributed to the absence of fabricated information in the belief-revision-grounded graph.
- **Automated AGM compliance testing**: 100% operational adherence to basic AGM postulates (K*2–K*6) and Hansson's postulates across 49 scenarios.

## 4. Limitations & Gaps
- Formal results are limited to simple propositional logic over ground triples—a deliberate trade-off of expressiveness for tractability.
- Supplementary AGM postulates (K*7, K*8) remain open questions requiring further formal development.
- The dual-store model (Redis + Neo4j) adds infrastructure complexity compared to single-store systems.
- As a single-author industry paper (Kumiho Inc.), independent replication of results is pending.
