# Google Developers Blog: Gemini 1.5 Pro 2M context

> Official blog | 2024-06

## 1. Core Claim
- Gemini 1.5 Pro introduced a 2 million token context window with context caching in the Gemini API, enabling cost reduction for repeated prompts and supporting code execution capabilities within the context window.

## 2. Method / Mechanism
- **2M token context window**: Processes up to 2 million tokens in a single prompt—equivalent to ~30 novels or 12 hours of video.
- **Context caching**: API feature that caches frequently used context tokens to reduce cost and latency on repeated queries.
- **Code execution**: Built-in code execution capabilities within the model's context window.
- **Mixture-of-Experts (MoE)**: Sparse expert activation reduces compute costs for extended contexts.

## 3. Key Evidence
- **2 million tokens** maximum context for Gemini 1.5 Pro.
- Context caching reduces cost for repeated prompts against the same long document.
- Joint vision-language embeddings with retrieval-augmented compression enable multimodal long-context processing.

## 4. Limitations & Gaps
- Source is a 404 page; details come from reading-log summary only.
- No data on actual latency for 2M token prompts or pricing after caching.
- "Lost in the middle" phenomenon (cited in S050) suggests even 2M tokens may not guarantee full information utilization.
- No discussion of memory architecture beyond raw context length—this is a long-context solution, not a structured memory system.
