# arXiv: Context Cartography (2603.20578)

> Academic paper | 2026-03

## 1. Core Claim
- Context windows alone are insufficient for LLM reasoning; a formal "Context Cartography" framework is needed to deliberately govern how information moves between sensing, memory, and active reasoning surfaces.

## 2. Method / Mechanism
- **Tripartite Zonal Model**:
  - **Black Fog**: Unobserved information (outside context window).
  - **Gray Fog**: Stored memory (retrievable but not currently active).
  - **Visible Field**: Active reasoning surface (current context window).
- **Seven Cartographic Operators**: Reconnaissance, Selection, Simplification, Aggregation, Projection, Displacement, and Layering — derived from systematic coverage of all non-trivial zone transformations.
- **Organization**: Operators classified by transformation type (what they do) and zone scope (where they apply).
- **Salience Geometry**: Grounded in transformer attention patterns, characterizing operators as compensations for linear prefix memory, append-only state, and entropy accumulation.

## 3. Key Evidence
- Analysis of four contemporary systems (Claude Code, Letta, MemOS, OpenViking) as interpretive evidence that cartographic operators are converging independently across industry.
- Derives testable predictions including operator-specific ablation hypotheses.
- Proposes a diagnostic benchmark for empirical validation.

## 4. Limitations & Gaps
- Framework is primarily theoretical/formal; empirical validation via the proposed benchmark is future work.
- Analysis of existing systems is interpretive rather than experimental — authors map observed behaviors to operators without controlled studies.
- No quantitative performance claims yet.

## 5. Relevance to Our Review
- Provides a **unifying theoretical lens** for evaluating memory systems across OpenViking, Claude Code, Mem0, and Honcho. Useful for the taxonomy/framework subsection of the review.
