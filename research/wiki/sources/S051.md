# Microsoft TechCommunity: Introducing Copilot Memory

> Official blog | 2025-12

## 1. Core Claim
- Microsoft introduced Copilot Memory as a generally available feature (July 2025), making persistent user context the default experience while providing both user and tenant admin controls over what is stored and how it is used.

## 2. Method / Mechanism
- **Intent-driven storage**: Memories are extracted and stored based on inferred user intent rather than raw transcript logging.
- **User controls**: Individuals can view, edit, and delete stored memories; turn memory features on or off.
- **Tenant admin controls**: Enterprise administrators can govern memory settings at the organizational level.
- **Default-on**: Memory is enabled by default for Copilot, accelerating adoption but raising privacy considerations.

## 3. Key Evidence
- **GA July 2025**: Memory in Copilot went generally available.
- **On by default**: Users must proactively disable if they do not want memory functionality.
- **Not available for Agents currently**: Memory is scoped to direct Copilot interactions, not extendable to custom agents built on Copilot Studio.

## 4. Limitations & Gaps

- "Intent-driven" is vague—unclear if this uses LLM-based extraction, heuristic rules, or hybrid approach.
