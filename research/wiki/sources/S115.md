# Memory-R1: Enhancing LLM Agents to Manage Memories via RL

> Academic paper | 2025-08

## 1. Core Claim
- Most memory-augmented LLM agents use static, heuristic-driven pipelines that lack a learned mechanism for deciding what to store, update, or retrieve.
- Memory-R1 uses reinforcement learning (PPO/GRPO) to train specialized Memory Manager and Answer Agent agents, enabling adaptive memory management with only 152 training QA pairs.

## 2. Method / Mechanism
- **Memory Manager agent**: Learns structured memory operations—ADD, UPDATE, DELETE, NOOP.
- **Answer Agent**: Pre-selects and reasons over relevant memory entries before generating responses.
- **Outcome-driven RL**: Both agents fine-tuned with PPO and GRPO using minimal supervision.
- **Minimal data requirement**: Only 152 training QA pairs needed for strong performance.

## 3. Key Evidence
- Outperforms strong baselines across **LoCoMo, MSC, and LongMemEval** benchmarks.
- Generalizes across diverse question types and multiple model scales (**3B–14B**).
- **LLaMA3.1-8B + GRPO** achieves F1 52.87, B1 37.54, J 30.64 on LoCoMo (vs. baseline F1 47.82).
- GRPO shows stronger gains than PPO in ablation comparisons.

## 4. Limitations & Gaps
- 152 training pairs, while impressive for minimal supervision, may limit robustness on out-of-distribution tasks or novel domains.
- The operation set {ADD, UPDATE, DELETE, NOOP} is minimal; richer operations (MERGE, SPLIT, LINK) are not explored.
- RL training instability may affect reproducibility, especially with small model scales.
- Evaluation focused on conversational benchmarks; task-oriented or tool-use agents are not tested.

## 5. Relevance to Our Review
- **Academic memory architectures**: A leading example of RL-based memory management.
- **RL-based memory**: Demonstrates that learned memory policies can outperform hand-designed heuristics with minimal data.
