# arXiv: OpenClaw case study / tri-layered risk

> Academic paper | 2026-03-13

## 1. Core Claim
OpenClaw's **tri-layered security model** has critical vulnerabilities: plaintext secrets stored in Markdown/SQLite memory files (CVE-2026-25253, CVSS 8.8–9.8), insecure state storage exposing authentication tokens, and a gateway authentication flaw enabling WebSocket URL manipulation.

## 2. Method / Mechanism
- **CVE-2026-25253**: Critical vulnerability in OpenClaw's memory persistence layer.
- **Plaintext secrets in Markdown/SQLite**: Sensitive credentials, API keys, and tokens are stored without encryption in MEMORY.md, daily notes, and the SQLite index.
- **Gateway auth flaw**: WebSocket gateway URL manipulation allows token exfiltration.
- **CVSS scoring**: 8.8–9.8 (critical severity).
- **Insecure state storage**: Session state and authentication material persisted locally without encryption at rest.

## 3. Key Evidence
- CVE-2026-25253 with CVSS 8.8–9.8.
- Plaintext storage confirmed in Markdown and SQLite files.
- Gateway authentication bypass via WebSocket URL manipulation.
- One-click RCE possibility (per S031 Skywork deep-dive).
