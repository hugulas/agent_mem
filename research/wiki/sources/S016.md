# CrabTalk: Hermes Agent survey

> Technical survey | 2026-03-14

## 1. Core Claim
Hermes Agent is a **Python-based open-source agent runtime** (MIT license, 6K GitHub stars in first month) positioned "between a Claude Code style CLI and an OpenClaw style messaging platform agent," featuring persistent skill documents, multi-level memory, and the agentskills.io portability standard.

## 2. Method / Mechanism
- **Model stack**: Hermes 3 (Llama 3.1 fine-tune, 8B/70B/405B) and Hermes 4 (hybrid reasoning with <think> tags, DataForge synthetic data generation, ~5M samples / ~60B tokens). Hermes 4.3 (36B) fine-tuned on ByteDance Seed 36B.
- **Training**: Atropos distributed RL framework with rejection sampling across ~1,000 task-specific verifiers.
- **ReAct loop**: Observation → Reasoning → Action → Loop, with innovation in surrounding systems rather than the loop itself.
- **Five memory layers**:
  1. Short-term inference memory (transformer context)
  2. Procedural skill documents (persistent markdown)
  3. Contextual persistence (searchable vector store for skills)
  4. User modeling via Honcho (entity-centric, peer-based)
  5. FTS5 full-text search (SQLite, cross-session recall)
- **Six execution backends**: Local, Docker, SSH, Daytona, Singularity, Modal — unified via BaseEnvironment interface.
- **MCP support**: Built-in client with exponential backoff reconnection.
- **agentskills.io**: Portable skill standard compatible with 11+ tools.
- **Local inference**: Ollama, vLLM, llama.cpp support.

## 3. Key Evidence
- 6,000 GitHub stars in first month.
- Hermes 2 Pro achieved 90% function calling accuracy vs 60–70% for general models.
- Hermes 4: 5× samples, 50× tokens vs Hermes 3.
- Hermes 4.3: 78.4% reduction in overlong reasoning on AIME'24, 4.7% accuracy cost.
- Six backends with single-line configuration.
