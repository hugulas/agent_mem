## 1. Core Claim
Three major coding agents — Codex CLI, Claude Code, and OpenCode — each implement context compaction with **qualitatively different tiering strategies**: Codex uses a single-layer "handoff summary," Claude Code deploys a 3-tier pipeline (trim/cache/summary) leveraging Prompt Cache, and OpenCode uses a 2-tier approach (hide/summary).

## 2. Method / Mechanism
- **Codex CLI**: Single-layer "handoff summary" — previous session summarized into a compact form passed to the next session.
- **Claude Code**: 3-tier pipeline:
  - Trim: remove oldest messages
  - Cache: use Anthropic Prompt Cache to preserve recent context cheaply
  - Summary: semantic compression of cached content when cache also overflows
- **OpenCode**: 2-tier (hide/summary) — hide less relevant context from immediate view, summarize when hiding is insufficient.

## 3. Key Evidence
- Side-by-side comparison of three tools on the same 15.4K token input (per S017).
- Codex: simplest approach, lowest complexity.
- Claude Code: most elaborate, vendor-specific (Prompt Cache).
- OpenCode: middle ground, 2-tier balancing simplicity and retention.
