# arXiv: Dive into Claude Code (2604.14228)

> Academic paper | 2026-04-14

## 1. Core Claim
Claude Code's architecture is built around a **minimal-scaffolding, maximal-operational-harness** philosophy: a simple while-loop for the agent core, with the vast majority of code (~98.4%) invested in surrounding infrastructure — a five-layer compaction pipeline, a seven-mode permission system with ML classifier, and four extensibility mechanisms (MCP, plugins, skills, hooks).

## 2. Method / Mechanism
- **Agent loop**: Simple `queryLoop()` async generator — model call → tool dispatch → result collection → repeat.
- **Five-layer compaction pipeline** (executes before every model call):
  1. Budget reduction (per-tool-result size limits)
  2. Snip (temporal depth trimming)
  3. Microcompact (cache overhead reduction via cache_edits API)
  4. Context collapse (very long history management)
  5. Auto-compact (semantic compression as last resort)
- **CLAUDE.md hierarchy**: Four-level instruction system from managed settings to directory-specific files, plus auto-memory entries.
- **Permission system**: Seven modes + ML-based auto-mode classifier (two-stage fast-filter + chain-of-thought evaluation).
- **Extensibility**: MCP (8+ transport variants), plugins, skills, hooks (27 event types).
- **Subagent delegation**: Spawns isolated context windows, returns summary-only to parent.
- **Context as binding constraint**: 200K (older) / 1M (Claude 4.6) token window. Per-tool-result budget (50K/200K chars), deferred tool schemas, lazy CLAUDE.md loading.

## 3. Key Evidence
- Source-level analysis of TypeScript codebase.
- Anthropic internal survey: 27% of tasks would not have been attempted without Claude Code; 93% approval rate for permission prompts.
- Auto-approve rates increase from ~20% (<50 sessions) to >40% (750+ sessions) — trust trajectory.
- Five-layer compaction pipeline with specific cost-benefit ordering (cheaper layers first).
- Comparison with OpenClaw across 6 design dimensions (Section 10).

## 4. Limitations & Gaps
- Authors note limited mechanisms for **long-term human capability preservation** — short-term amplification may come at cost of sustained codebase coherence.
- Study finds developers in AI-assisted conditions score 17% lower on comprehension tests ("paradox of supervision").
- Analysis based on publicly available sources only; no access to private Anthropic docs.
- No quantitative benchmarks on compaction quality or recall accuracy.
