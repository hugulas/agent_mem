# ClawTank: OpenClaw + Supermemory setup
> **STATUS: HOMEPAGE NOT ARTICLE** — PDF is ClawTank homepage, not setup guide. The content in this note may be incomplete or reconstructed from reading-log summaries.

> Integration guide | 2025-11-07

## 1. Core Claim
- Supermemory transforms OpenClaw from a stateless Markdown-based chatbot into a cloud-powered personal assistant with semantic memory, auto-capture, and knowledge graph visualization.

## 2. Method / Mechanism
- **Plugin**: `@supermemory/openclaw-supermemory` installed via `openclaw plugins install`.
- **Auto-capture**: Every conversation stored automatically after each message.
- **Smart recall**: Relevant memories injected as context before each AI turn.
- **Semantic search**: Meaning-based retrieval, not just keyword matching.
- **Cross-session persistence**: Memory tied to user account (via `containerTag`), not container — survives rebuilds.
- **Knowledge graph visualization**: Interactive dashboard showing memory nodes, connections, and timeline.
- **Setup paths**: Self-hosted (Pro $29/mo) or pre-configured on ClawTank cloud deploy.

## 3. Key Evidence
- Step-by-step integration guide with exact commands.
- Memory isolation handled per-user via `containerTag`.
- On ClawTank, plugin is auto-installed during container provisioning.

## 4. Limitations & Gaps
- Requires Supermemory Pro ($29/mo) — no free integration tier.
- Cloud-dependent; if Supermemory service is down, OpenClaw loses its long-term memory.
- OpenClaw's built-in memory is characterized as "limited" and "basic," which contradicts S001/S002 describing hybrid vector+BM25 search and multiple backends — this is likely marketing bias.
- No performance benchmarks or latency measurements for the integration.
