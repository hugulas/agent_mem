# Codex CLI system requirements (deployhq)

> Official-ish guide | 2026-04-11

## 1. Core Claim
Codex CLI has modest system requirements: Node.js 22+, native macOS/Linux support with Windows via WSL, 4 GB minimum RAM and 8 GB recommended, and a lightweight ~50 MB package that also offers a Rust binary alternative.

## 2. Method / Mechanism
- Node.js 22+ runtime dependency ( aligns with latest LTS )
- Cross-platform packaging: native macOS/Linux, Windows through WSL layer
- Dual distribution: standard JS package (~50 MB) plus optional Rust binary for potentially faster execution
- Deployment via standard Node.js package managers (npm/yarn)

## 3. Key Evidence
- Node.js 22+ required
- 4 GB RAM minimum / 8 GB recommended
- Package size: ~50 MB
- Rust binary available as alternative to Node.js runtime

## 4. Limitations & Gaps
- Windows WSL requirement implies no native Windows support (S040 contradicts this with Windows 10 native install info)
- No mention of disk I/O requirements for the memory subsystem (AGENTS.md + SQLite)
- No quantified performance difference between Node.js and Rust binary builds
- Memory system specs (AGENTS.md 32 KiB ceiling) not discussed
