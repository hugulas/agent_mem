# ACON: Optimizing Context Compression for Long-Horizon LLM Agents

> Academic paper | 2025-10

## 1. Core Claim
- Prior context compression work has focused on single-step tasks or narrow applications; long-horizon agents need compression optimized for multi-step trajectories.
- Agent Context Optimization (ACON) learns compression guidelines from paired failure/success trajectories and can be distilled into smaller models, achieving substantial token reduction with minimal accuracy loss.

## 2. Method / Mechanism
- **Compression guideline optimization in natural language space**: Given paired trajectories where full context succeeds but compressed context fails, a capable LLM analyzes failure causes and updates the compression guideline.
- **Iterative refinement**: Guidelines are refined over multiple failure-analysis rounds.
- **Distillation**: The optimized LLM compressor is distilled into smaller models to reduce inference overhead.
- **Unified framework**: Compresses both environment observations and interaction histories.

## 3. Key Evidence
- **AppWorld, OfficeBench, Multi-objective QA**: Reduces peak token usage by 26–54%.
- Preserves over 95% of task accuracy when distilled into smaller compressors.
- Enhances smaller LMs (e.g., Qwen3-14B distilled) as long-horizon agents with up to 46% performance improvement.
- Accuracy–peak-token trade-offs show ACON improves accuracy on smaller models while reducing cost on larger ones.

## 4. Limitations & Gaps
- Guideline optimization requires collecting paired success/failure trajectories, which is expensive and task-dependent.
- Distillation quality depends heavily on the teacher compressor; domain shift may degrade distilled performance.
- The framework adds an extra compression module to the agent pipeline, introducing latency unless optimized.
- Evaluation is on app-use and office benchmarks; generalization to coding or creative writing agents is not demonstrated.

## 5. Relevance to Our Review
- **Context compression**: A leading example of learned guideline-based compression for long-horizon agents.
- **Benchmarks**: Strong quantitative results on realistic agent benchmarks (AppWorld, OfficeBench).
