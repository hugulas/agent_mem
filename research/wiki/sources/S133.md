# Ethiack: One-Click RCE on OpenClaw (Moltbot)

> Security analysis | 2026-03

## 1. Core Claim
Ethiack's autonomous hacking agent **Hackian** discovered a critical one-click RCE vulnerability in OpenClaw (aka Moltbot) in **under 2 hours**. The vulnerability (CVE-2026-25253) allows an attacker to hijack an OpenClaw instance by tricking a user into clicking a malicious link containing a crafted `gatewayUrl` parameter.

## 2. Method / Mechanism
- **Discovery**: Autonomous AI pentest robot (Hackian) found the vulnerability autonomously
- **Attack mechanism**:
  1. OpenClaw Control UI accepts `gatewayUrl` from URL query parameters
  2. UI automatically establishes WebSocket connection to the supplied URL without user confirmation
  3. Auth token from `localStorage` is sent to the attacker-controlled WebSocket endpoint
  4. Attacker uses stolen token to impersonate victim and execute commands
- **Localhost bypass**: Even locally deployed instances (127.0.0.1) are vulnerable because browsers allow WebSocket connections to localhost with fewer restrictions
- **Fix**: Commit `8cb0fa993ba0ba4a3e24c2a9e0416ab2ca796dcf` separates instruction memory from data memory

## 3. Key Evidence
- Hackian discovered the bug in <2 hours autonomously
- CVE-2026-25253 assigned
- Official fix commit published on GitHub
- Affects all versions before v2026.1.29

## 4. Limitations & Gaps
- Ethiack is a commercial security firm; report may emphasize the severity for marketing
- No independent third-party verification of the "<2 hours" claim
- Focuses on the sensational aspect (AI-discovered bug) rather than deep technical analysis
