## 1. Core Claim
- Hermes's Honcho integration transmits full conversation text verbatim to api.honcho.dev and runs third-party LLM inference on both user and agent messages without adequate disclosure, creating a significant privacy gap relative to the project's "all data stays on your machine" claims.

## 2. Method / Mechanism
- **Data transmitted to api.honcho.dev**:
  1. Bidirectional inference: Both peers registered with `observe_me=True`; Honcho backend runs its own LLM on both sides.
  2. Full message text: Every user and assistant message synced verbatim, not summaries or embeddings.
  3. Peer identity: User name, workspace ID, session key derived from working directory.
  4. Local memory files: `MEMORY.md`, `USER.md`, `SOUL.md` uploaded during migration.
  5. Full conversation history: Entire log uploaded as XML transcript during migration.
  6. Model-generated conclusions: Remote storage of inferred user preferences.
- **Background prefetch threads**: Send context and dialectic requests between turns without visible indicator.
- **Setup disclosure gap**: Wizard describes Honcho as "persistent cross-session memory" without stating what data is transmitted or how it is processed remotely.
