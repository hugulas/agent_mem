# Hermes docs: Memory Providers

> Official docs | 2026-05

## 1. Core Claim
Hermes Agent ships with **8 external memory provider plugins** that add persistent, cross-session knowledge beyond the always-active built-in MEMORY.md and USER.md. Only one external provider can be active at a time, integrated via a unified setup wizard.

## 2. Method / Mechanism
- **8 external providers**: Honcho, OpenViking, Mem0, Hindsight, Holographic, RetainDB, ByteRover, Supermemory.
- **Built-in memory always active**: MEMORY.md + USER.md continue working regardless of external provider selection.
- **Six automatic behaviors** when provider active:
  1. Inject provider context into system prompt
  2. Prefetch relevant memories before each turn (background, non-blocking)
  3. Sync conversation turns to provider after each response
  4. Extract memories on session end (for supporting providers)
  5. Mirror built-in memory writes to external provider
  6. Add provider-specific tools
- **Setup**: `hermes memory setup` interactive wizard; `hermes memory status`; `hermes memory off`.
- **Honcho integration details**: Two-layer context injection (base layer + dialectic supplement), 3 config knobs (contextCadence, dialecticCadence, dialecticDepth), multi-peer workspace model.
- **OpenViking**: Tiered loading L0 (~100 tokens) → L1 (~2K) → L2 (full), viking:// URI scheme.
- **Hindsight**: Knowledge graph with reflect operation, LongMemEval 91.4% (Gemini-3).
- **Holographic**: Local SQLite, zero pip dependencies, HRR algebra, 9 fact_store actions.

## 3. Key Evidence
- 8 providers documented with specific tools, storage, cost, and best-for use cases.
- Honcho: 5 tools (profile, search, context, reasoning, conclude).
- Hindsight: LongMemEval 91.4% with Gemini-3.
- Holographic: zero external dependencies.
- OpenViking: L0/L1/L2 tiered retrieval.

## 4. Limitations & Gaps
- Only one external provider at a time — cannot combine strengths of multiple providers (e.g., Hindsight's graph + Mem0's semantic search).
- Honcho AGPL v3.0 license creates copyleft concerns for commercial self-hosting.
- Prefetching is background/non-blocking but adds latency to first response if cache miss.
- No quantitative comparison of provider latency or resource usage within Hermes.
