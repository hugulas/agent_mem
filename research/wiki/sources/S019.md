# OpenClaw Hardware (sfailabs)

> Deployment guide | 2026-04-14

## 1. Core Claim
OpenClaw hardware requirements scale **dramatically by use case**: text-only solo operation needs just 2 cores / 4 GB, but browser automation demands 4 cores / 8 GB, multi-agent setups need 4–8 cores / 16 GB, and air-gapped local LLM deployment requires 8+ cores / 32 GB plus GPU.

## 2. Method / Mechanism
- **Text-only solo**: 2 cores / 4 GB RAM.
- **Browser automation**: 4 cores / 8 GB RAM.
- **Multi-agent**: 4–8 cores / 16 GB RAM.
- **Air-gapped local LLM**: 8+ cores / 32 GB RAM + GPU.

## 3. Key Evidence
- Scenario-based tiering with specific core/RAM pairs.
- GPU explicitly required only for local LLM deployment.
- Multi-agent scaling shows sub-linear resource growth (4–8 cores for multi-agent vs 2 for solo).

## 4. Limitations & Gaps
- **Source is a 404 page**. Content from reading-log key claims only.
- No GPU VRAM specifications (critical for local LLM viability).
- No disk space guidance for SQLite indices, embedding caches, or model weights.
- No network requirements for messaging platform bridges.
- No benchmark data validating these recommendations under actual load.
