# 万字拆解OpenClaw (huxiu)

> Chinese deep-dive | 2026-03-16

## 1. Core Claim
OpenClaw implements a pre-compaction memory flush triggered at a 4,000-token soft threshold (or 2 MB transcript threshold), designed as a "memory moat" (护城河) to protect context window integrity before expensive compaction operations occur.

## 2. Method / Mechanism
- Pre-compaction flush: proactive memory write-before-compaction to prevent context overflow
- Dual triggers: 4,000 token soft threshold AND 2 MB transcript size threshold
- "Memory moat" design philosophy: defensive architecture that prioritizes preserving user context over aggressive compaction
- Operates before the more expensive compaction pipeline kicks in

## 3. Key Evidence
- Memory flush soft threshold: 4,000 tokens
- Transcript size threshold: 2 MB
- Design described as "memory护城河" (memory moat)
- Connects to S006's SNIPPET_MAX_CHARS=700 and DEFAULT_MEMORY_FLUSH_SOFT_TOKENS=4000 constants

## 4. Limitations & Gaps
- No disclosed benchmark on flush frequency in real workloads
- 4,000 token threshold is relatively low for modern models with 128K+ context windows
- No discussion of whether flush is synchronous (blocking) or asynchronous
- 2 MB transcript threshold may correlate poorly with token count for non-English text
- Source is a Chinese media article; translation and technical accuracy may vary
