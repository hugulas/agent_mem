# Mem0 GitHub: New Memory Algorithm April 2026

> Official repo | 2026-04

## 1. Core Claim
- Mem0's April 2026 algorithm achieves significant benchmark improvements through single-pass ADD-only extraction, entity linking, temporal reasoning, and multi-signal retrieval — establishing a new production baseline for agent memory.

## 2. Method / Mechanism
- **Single-pass ADD-only extraction**: One LLM call per extraction; memories accumulate without overwriting — simpler pipeline than UPDATE/DELETE.
- **Agent-generated facts as first-class**: Confirmed agent actions stored with equal weight to extracted observations.
- **Entity linking**: Entities extracted, embedded, and linked across memories for retrieval boosting.
- **Multi-signal retrieval**: Semantic search, BM25 keyword matching, and entity matching scored in parallel and fused.
- **Temporal reasoning**: Time-aware retrieval ranks the correct dated instance for queries about current state, past events, and upcoming plans.
- **Deployment options**: Library (`pip install mem0ai`), self-hosted server (Docker Compose), managed cloud (app.mem0.ai).

## 3. Key Evidence
- **Benchmark results**:
  - LoCoMo: 91.6 (+20.2 over old algorithm at 71.4)
  - LongMemEval: 94.8 (+27.0 over old algorithm at 67.8)
  - BEAM 1M: 64.1
  - BEAM 10M: 48.6
- **Latency**: p50 ~0.88s–1.09s across benchmarks.
- **Token usage**: ~6.7K–7.0K tokens per query.
- All benchmarks run on same production-representative model stack with single-pass retrieval (no agentic loops).

## 4. Limitations & Gaps
- ADD-only risks memory bloat over long horizons; no compaction or deletion mechanism described.
- BEAM 10M score (48.6) shows degradation at very large scales.
- Promotion material from Mem0 itself; independent verification desirable.
- No direct comparison with Hindsight or ByteRover on this page.
