# Claude Code Architecture Analysis (bits-bytes-nn)

> Technical analysis | 2026-03-31

## 1. Core Claim
Comprehensive source-level analysis of Claude Code's leaked TypeScript codebase (~4,600+ files), covering the 5-layer compaction pipeline, tool system, permission architecture, and context management.

## 2. Method / Mechanism
- **5-layer compaction**: Snip → Microcompact (cache-aware pinning) → Context Collapse → Auto-Compact (contextWindow - 13_000 buffer) → Session Memory Compact
- **Tool system**: ~20 base tools + conditionally active + feature-flag-gated unreleased tools
- **Tool budgets**: 50K chars per tool result, 200K chars per message; definitions consume ~55K tokens in system prompt
- **Auto-compact threshold**: `effectiveContextWindow - 13_000` tokens; circuit breaker at 3 consecutive failures
- **Microcompact**: Clears stale tool results when cache TTL (60 min) expired; preserves prompt cache hit rates

## 3. Key Evidence
- Auto-compact code: `return effectiveContextWindow - 13_000`
- Circuit breaker: `MAX_CONSECUTIVE_AUTOCOMPACT_FAILURES = 3`
- Tool definitions: "tools alone consume thousands of tokens in the system prompt" (~55K total)
- Microcompact pinning strategy documented with code snippets

## 4. Limitations & Gaps
- Based on leaked source code analysis; claims may not reflect current production version
- Some features (WebBrowserTool, WorkflowTool, SleepTool) are feature-flag-gated and unreleased
