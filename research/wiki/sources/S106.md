# MAGMA: Multi-Graph Based Agentic Memory Architecture

> Academic paper | 2026-01

## 1. Core Claim
- Monolithic memory stores that rely on semantic similarity entangle temporal, causal, and entity information, limiting interpretability and alignment between query intent and retrieved evidence.
- Decoupling memory representation into orthogonal semantic, temporal, causal, and entity graphs—combined with policy-guided traversal—enables query-adaptive retrieval and transparent reasoning paths.

## 2. Method / Mechanism
- **Multi-graph architecture**: Each memory item is represented across four orthogonal graphs (semantic, temporal, causal, entity).
- **Policy-guided retrieval**: Formulates retrieval as traversal over relational views, enabling query-adaptive selection and structured context construction.
- **Decoupled design**: Memory representation is separated from retrieval logic, providing fine-grained control and transparent reasoning paths.

## 3. Key Evidence
- Outperforms state-of-the-art agentic memory systems on **LoCoMo** and **LongMemEval** benchmarks.
- Numeric data shows latency/accuracy trade-offs against baselines (A-MEM, MemoryOS, Nemori).
- Demonstrates consistent improvements in long-horizon reasoning tasks where multi-relational reasoning is required.

## 4. Limitations & Gaps
- Maintaining four separate graphs increases storage overhead and system complexity compared to monolithic approaches.
- Policy-guided traversal requires learning or tuning traversal policies, adding a training burden.
- The decoupling benefit assumes queries clearly benefit from one type of relation over others; mixed-intent queries may complicate policy selection.
