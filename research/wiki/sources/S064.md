# Red Hat: Deploy OpenViking on OpenShift AI

> Technical deployment guide | 2026-04

## 1. Core Claim
- OpenViking's tiered context loading (L0/L1/L2) achieves substantial token savings compared to flat RAG when deployed in production environments like OpenShift AI.

## 2. Method / Mechanism
- **Deployment Target**: OpenShift AI with REST API, Python SDK, and CLI interfaces.
- **Tiered Loading**: L0 (~100 tokens), L1 (~2K tokens), L2 (full document) loaded on demand.
- **Token Savings Example**: 50 runbooks — flat RAG consumes ~50K tokens vs. OpenViking ~5K (L0) + ~6K (L1) + one L2 document.

## 3. Key Evidence
- Quantified token savings math: ~50K tokens (RAG) vs. ~11K + one L2 (OpenViking) for 50-runbook scenario.
- Production deployment architecture via OpenShift AI.
