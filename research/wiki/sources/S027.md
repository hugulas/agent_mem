# Regolo: Benchmark memory usage

> Benchmark blog | 2026-04-27

## 1. Core Claim
OpenClaw exhibits dramatically worse memory system performance than Hermes under a 300-event benchmark: recall latency of 19,593 ms vs 113 ms (173× slower) and a disk delta of 213 KB vs 0 KB, indicating OpenClaw's memory writes data to disk while Hermes operates purely in-memory.

## 2. Method / Mechanism
- 300-event synthetic workload benchmark
- Two metrics measured: recall latency and disk write delta
- Controlled comparison between OpenClaw and Hermes memory subsystems
- Disk delta measured to detect persistent storage side effects during recall

## 3. Key Evidence
- Recall latency: OpenClaw 19,593 ms vs Hermes 113 ms
- Disk delta: OpenClaw 213 KB vs Hermes 0 KB
- Test scale: 300 events
- Implication: OpenClaw's SQLite-backed memory incurs I/O overhead; Hermes's in-memory approach is faster but potentially volatile

## 4. Limitations & Gaps
- No disclosure of hardware specs or concurrency level
- 300 events may not represent production workloads
- Disk delta measurement method is unclear (is it per-query or cumulative?)
- Single benchmark dimension; misses accuracy, consistency, or multi-session tests
- Source (Regolo.ai) may have commercial relationship with one or both platforms

## 5. Relevance to Our Review
One of the most critical benchmark sources for the "Benchmarks" section, providing quantified evidence of OpenClaw's memory latency problem.
