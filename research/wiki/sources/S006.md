# MMNTM: How OpenClaw Implements Agent Memory

> Code-level analysis | 2026-02-01

## 1. Core Claim
OpenClaw's agent memory system makes **pragmatic trade-offs that work**: Markdown files as the canonical source of truth, a hybrid search combining vectors with keywords via union (not intersection), and a clever pre-compaction flush that saves memories before context overflow destroys them.

## 2. Method / Mechanism
- **Files as source of truth**: MEMORY.md (curated long-term facts), memory/YYYY-MM-DD.md (append-only daily logs), and {agentId}.sqlite (derived index for semantic search). "The index is derived, the files are canonical."
- **Hybrid search (union)**: 70% vector + 30% keyword. BM25 ranks converted to scores via `1 / (1 + normalized_rank)`. Keyword search uses AND logic on quoted tokens.
- **Candidate over-fetching**: Default multiplier of 4× (request 6 results → fetch 24 from each search).
- **Graceful degradation**: Each search wrapped in `.catch(() => [])` — if embeddings fail, keyword search still works.
- **Pre-compaction flush**: Before context compaction, an automatic agentic turn prompts the model to save durable memories. Triggered at soft threshold of 4,000 tokens (`DEFAULT_MEMORY_FLUSH_SOFT_TOKENS`).
- **Embedding provider priority**: Local GGUF → OpenAI → Gemini. Falls back to BM25-only if all fail.
- **Source-level constants**: `SNIPPET_MAX_CHARS=700`, `EMBEDDING_BATCH_MAX_TOKENS=8000`, `EMBEDDING_INDEX_CONCURRENCY=4`.

## 3. Key Evidence
- Direct source code citations from `src/memory/hybrid.ts`, `src/memory/manager.ts`, `src/auto-reply/reply/memory-flush.ts`.
- Default soft threshold: 4,000 tokens for pre-compaction memory flush.
- Candidate multiplier: 4× over-fetch.
- Source constants documented: SNIPPET_MAX_CHARS=700, EMBEDDING_BATCH_MAX_TOKENS=8000, EMBEDDING_INDEX_CONCURRENCY=4, DEFAULT_MEMORY_FLUSH_SOFT_TOKENS=4000.

## 4. Limitations & Gaps
- Keyword search uses strict AND logic — "commit hash" requires both words; won't find partial matches.
- Pre-compaction flush depends on the model's judgment of what's "important" — may miss critical details.
- SQLite as derived index means large memory corpora may face scaling limits.
- No quantitative evaluation of recall accuracy vs. pure vector or pure BM25 baselines.
