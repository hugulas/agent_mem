# RetainDB features page

> Official docs | 2026-05

## 1. Core Claim
- RetainDB is a production-ready "memory + context brain" for AI agents, combining hybrid search, 7 typed memory systems, delta compression, and knowledge graphs into a single managed platform.

## 2. Method / Mechanism
- **Hybrid retrieval**: Vector + BM25 + reranking pipeline.
- **7 memory types**: Factual, preference, event, relationship, opinion, goal, instruction — each with temporal reasoning and version chains.
- **Delta compression**: 50–90% token savings in multi-turn conversations by sending only what changed.
- **Oracle Research Mode**: Tree-guided multi-step autonomous search for complex questions.
- **Knowledge graphs**: Auto-extracted entities and relationships.
- **15+ connectors**: GitHub, Notion, Slack, Discord, Confluence, PostgreSQL, MongoDB, etc.
- **TypeScript SDK**: 40+ production-ready methods.

## 3. Key Evidence
- Claims **<50 ms memory retrieval**, **100% memory retention**, and **99.9% uptime SLA**.
- Marketing claims **$300–$2,000/month LLM cost savings** from delta compression.
- Auto-extraction uses Claude Sonnet 4.5 (per reading-log).

## 4. Limitations & Gaps
- All performance claims are self-reported without independent verification.
- Features page implies cloud-only; self-hostability (per S087) is not mentioned here.
- Credit card required even for free trial.
- "0% hallucination rate on documentation questions" (per S089) is suspiciously absolute.
