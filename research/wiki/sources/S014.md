# OpenWalrus: Hermes memory five layers

> Technical blog | 2026-03-15

## 1. Core Claim
Hermes Agent's memory system comprises **five distinct layers** with a clear cost and locality breakdown: three layers are local and free (context window, skills, FTS5), while two require external services (contextual vector, Honcho).

## 2. Method / Mechanism
- **5 layers**:
  1. Context window (ephemeral, per-session)
  2. Skills (procedural memory, local markdown in ~/.hermes/skills/)
  3. Contextual vector (optional, external provider)
  4. Honcho (optional, external user modeling)
  5. FTS5 full-text search (local SQLite)
- **3 layers local/free**: Context window + skills + FTS5 require no external services.
- **Character limits**: memory_char_limit=2200, user_char_limit=1375.
- **Skills as procedural memory**: Self-improving markdown documents capturing task approaches.

## 3. Key Evidence
- 5-layer breakdown with cost attribution.
- memory_char_limit=2200, user_char_limit=1375.
- 3 local/free layers vs 2 external layers.
