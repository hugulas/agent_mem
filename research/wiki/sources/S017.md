# Context Compaction comparison (justin3go)

> Benchmark blog | 2026-04-12

## 1. Core Claim
A **quantified side-by-side comparison** of context compaction across Codex CLI, Claude Code, and OpenCode using the same 15.4K token input reveals dramatically different tiering strategies: Codex uses a single-layer "handoff summary," Claude Code deploys a 3-tier pipeline with Prompt Cache, and OpenCode uses a 2-tier hide/summary approach.

## 2. Method / Mechanism
- **Same input**: 15.4K tokens fed to all three systems.
- **Codex CLI**: Single-layer "handoff summary" — previous context compacted into one summary passed forward.
- **Claude Code**: 3-tier pipeline (trim → cache → summary) leveraging Anthropic's Prompt Cache for efficient context preservation.
- **OpenCode**: 2-tier (hide → summary) — less relevant context hidden from view, then summarized when needed.

## 3. Key Evidence
- Identical 15.4K token input across all three tools.
- Codex: 1-layer compaction.
- Claude Code: 3-tier with Prompt Cache.
- OpenCode: 2-tier.
