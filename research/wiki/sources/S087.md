# RetainDB GitHub repo

> Official repo | 2026-04-03

## 1. Core Claim
- RetainDB is a self-hostable, open-source memory layer for AI agents that provides persistent, structured memory with typed relationships, temporal validity, and version chains.

## 2. Method / Mechanism
- **Deployment**: Docker Compose or Node.js + PostgreSQL + pgvector; single-tenant by default.
- **Memory graph**: Relations typed as `updates`, `contradicts`, `supports`, `derives`.
- **Temporal validity**: Facts carry `validFrom` / `validUntil`; stale knowledge is superseded, not deleted.
- **Retrieval**: One-call semantic search + BM25 + rerank, packed into a context string ready for the LLM.
- **MCP server**: Works with Claude Desktop and any MCP host out of the box.
- **Framework adapters**: Vercel AI SDK, LangChain, LangGraph.
- **SDK**: `@retaindb/sdk` v4.8.0 with 40+ methods.

## 3. Key Evidence
- Open-source under Apache 2.0 / BSL 1.1.
- Self-hosted local embedding fallback (no OpenAI key required).
- REST API with endpoints for memory, profile, search, session ingest, context query, and project management.

## 4. Limitations & Gaps
- Very early stage (only 8 GitHub stars per S089).
- OSS notes indicate some cloud features (e.g., Playwright connector) degrade to standard web crawler in self-hosted mode.
- No published benchmark results in the repo.
- Multi-tenant isolation requires cloud version; self-hosted is single-tenant with a shared server key.

## 5. Relevance to Our Review
- **External memory providers**: RetainDB as an emerging commercial/open-source hybrid option.
