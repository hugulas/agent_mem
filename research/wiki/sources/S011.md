# OpenAI Codex docs: Memories

> Official docs | 2026-05

## 1. Core Claim
OpenAI Codex CLI's memory system is configurable via TOML config keys under the `[memories]` section, with explicit controls for generation, usage, model selection, and geographic availability restrictions.

## 2. Method / Mechanism
- **Config structure**: `[memories]` section in `~/.codex/config.toml`.
- **Key toggles**:
  - `memories.generate_memories` — enable/disable background memory generation
  - `memories.use_memories` — enable/disable reading memories into context
  - `memories.extract_model` — model used for per-session extraction
  - `memories.consolidation_model` — model used for merging extracted memories
- **Geographic limits**: Memories feature excluded in EEA, UK, and Switzerland at launch.
- **Asymmetric modes**: Read-only (debugging), write-only (one-time onboarding), or full bidirectional.

## 3. Key Evidence
- Config keys documented in official repo (though specific page was 404 at download time).
- Geographic exclusion explicitly documented.
- Two-model pipeline (extract + merge) configurable independently.
