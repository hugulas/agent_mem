# ByteRover arXiv paper (2604.01599)

> Academic paper | 2026-04-02

## 1. Core Claim
- Same as S082: Agent-native memory architecture where the reasoning LLM also curates, structures, and retrieves knowledge, eliminating semantic drift from external-service memory pipelines.

## 2. Method / Mechanism
- Same as S082: Hierarchical Context Tree (Domain → Topic → Subtopic → Entry), Adaptive Knowledge Lifecycle (AKL), 5-tier progressive retrieval, zero external infrastructure.

## 3. Key Evidence
- Benchmark table shows multiple scores: ByteRover 93.2, Hindsight 89.9, Memobase 89.6, and others on various sub-metrics.
- State-of-the-art on LoCoMo; competitive results on LongMemEval.
- All knowledge stored as local Markdown files.

## 4. Limitations & Gaps
- Duplicate source of S082; same limitations apply.
- Limited to two benchmarks; no multi-modal or multi-agent evaluation.
- Authorship bias (ByteRover team evaluating their own system).

## 5. Relevance to Our Review
- **Academic memory architectures**: Agent-native curation and hierarchical context.
- **Benchmarks**: LoCoMo and LongMemEval performance data.
- **External memory providers**: ByteRover as a Hermes-integrated provider.
