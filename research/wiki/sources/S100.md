# OpenClaw docs: Memory overview & embedding providers

> Official docs | 2026-05-12

## 1. Core Claim
- OpenClaw uses a transparent, file-based memory system with no hidden state: MEMORY.md for durable facts, daily notes for working context, and optional DREAMS.md for human-reviewed dream summaries, plus 7 embedding provider options.

## 2. Method / Mechanism
- **Three memory files**:
  - **MEMORY.md**: Compact curated layer (durable facts, preferences, decisions); loaded at start of every session. Truncated if it exceeds bootstrap budget.
  - **memory/YYYY-MM-DD.md**: Working layer (daily notes, observations, session summaries); indexed for search but not auto-injected on every turn.
  - **DREAMS.md** (optional): Dream diary and sweep summaries for human review.
- **Memory tools**: `memory_search` (semantic search), `memory_get` (read specific file/range).
- **memory-wiki plugin**: Compiles durable knowledge into a wiki vault with provenance, contradiction tracking, freshness monitoring, and generated dashboards.
- **7 embedding providers**: OpenAI, Gemini, Voyage, Mistral, DeepInfra, Ollama, Local GGUF (~0.6 GB).
- **4 memory backends**: builtin SQLite, QMD, Honcho, LanceDB.
- **Commitments**: Opt-in, short-lived follow-up memories inferred in background and delivered via heartbeat.

## 3. Key Evidence
- File-based transparency: "The model only 'remembers' what gets saved to disk — there is no hidden state."
- Automatic truncation handling with diagnostic commands (`/context list`, `openclaw doctor`).
- Semantic search works "even when the wording differs from the original."

## 4. Limitations & Gaps
- File-based storage may not scale to massive corpora compared to dedicated vector databases.
- MEMORY.md truncation means not all long-term memory is always in context — signals a capacity ceiling.
- No published benchmark scores (LoCoMo, LongMemEval) for OpenClaw's native memory.
- DREAMS.md and commitments are quirky features that may not generalize to all agent use cases.

## 5. Relevance to Our Review
- **Academic memory architectures**: Transparent, human-readable memory as a design philosophy.
- **External memory providers**: OpenClaw supports Honcho and LanceDB as external backends; can also integrate Supermemory (S098).
