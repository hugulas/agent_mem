## 1. Core Claim
- Doubao phone (ByteDance's AI phone) employs a "cloud understanding + local storage + vector retrieval" architecture, where heavy inference runs in the cloud while screen context and embeddings are managed locally for low-latency retrieval.

## 2. Method / Mechanism
- **Cloud model**: InternVL3-2B serves as the cloud-based vision-language understanding model.
- **Local embedding models**: Run on-device to vectorize screen content and user context for fast retrieval.
- **Data flow**: Phone uploads ~250KB packets every 3–5 seconds (compressed screen frames + context) to cloud; cloud returns ~1KB action instruction packets.
- **Vector retrieval**: Local storage with vector indexing bridges cloud decisions and on-device execution.
