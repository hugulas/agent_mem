> Abstract
> To cope with the large contexts that long-horizon LLM agents produce, modern
> frameworks increasingly rely on compaction – invoking an LLM to rewrite the
> accumulated trajectory into a shorter summary that the agent resumes from. Today,
> compaction runs synchronously on the critical path of agent execution but this can
> unpredictably degrade accuracy due to a structural validation gap: the compactor
> must condense context but is fundamentally unaware of precisely what information
> the agent will need later. Further, because post-compaction agent steps are con-
> ditioned on the new summary, targeted validation criteria do not exist and errors
> silently propagate through coherent but incorrect behavior. Our key insight is that
> asynchronous compaction efficiently addresses this gap: by running the compactor
> in parallel with continued agent execution on the original context, the candidate
> summary and the agent’s next steps are generated independently from the same
> pre-compaction state, yielding a validation signal independent of the summary
> itself. We build Slipstream, a trajectory-grounded compaction system that uses a
> judge to validate the candidate summary against the agent’s continued reasoning,
> checking that it preserves both the agent’s forward intent and the key facts and
> constraints it depends on. Across long-horizon coding (SWE-bench Verified) and
> web-browsing (BrowseComp) workloads, Slipstream improves task accuracy by up
> to 8.8 percentage points while reducing end-to-end latency by up to 39.7%. We
> open source Slipstream at https://github.com/chenzhuofu/slipstream.

## 1. Core Claim
- Synchronous compaction suffers from a structural validation gap: the compactor must condense context without knowing what the agent will need later, causing silent accuracy degradation.
- Asynchronous compaction, running in parallel with continued agent execution on the original context, yields an independent validation signal by comparing the candidate summary against the agent's actual continued reasoning.

## 2. Method / Mechanism
- **Slipstream system**: Trajectory-grounded compaction validation.
- **Asynchronous execution**: Compactor runs in parallel with agent execution on original context; candidate summary and next steps are generated independently from the same pre-compaction state.
- **Trajectory-grounded judge**: Validates candidate summary against agent's continued reasoning with two checks:
  - *Plan-level*: Preserves agent's forward intent.
  - *Statement-level*: Preserves key facts and constraints.
- **Adopt/reject logic**: Accept compacted context if validation passes; continue with original if rejected (rejection rate: 1–8%).

## 3. Key Evidence
- **SWE-bench Verified** and **BrowseComp**: Improves task accuracy by up to 8.8 percentage points.
- Reduces end-to-end latency by up to 39.7% by hiding compaction latency in parallel execution.
- Documents that synchronous compaction inflates E2E execution times by 26–44% across mainstream workloads.
- Context rot causes accuracy drops of 14–85% as context grows.

## 4. Limitations & Gaps
- Requires infrastructure to run compactor and agent in parallel, which may not be feasible in resource-constrained or single-threaded deployments.
- The judge itself is an LLM call, adding some overhead even if hidden by parallelism.
- Rejection rate (1–8%) means occasional wasted compaction effort.
- Evaluated on coding and web-browsing agents; generalization to other long-horizon domains (e.g., GUI navigation) is not tested.

## 5. Relevance to Our Review
- **Context compression / compaction**: First rigorous validation of compaction via future agent trajectory.
- **Benchmarks**: Provides quantitative evidence on compaction overhead (26–44% E2E time inflation) that directly impacts production agent design.
