## 1. Core Claim
- Apple Intelligence uses a hybrid on-device/cloud architecture centered on a semantic index for personal context, an App Intents toolbox for cross-app actions, and an orchestration layer that routes queries between local models, Private Cloud Compute, and external AI services.

## 2. Method / Mechanism
- **Semantic index**: Organizes personal data (messages, emails, photos, calendar events) into a searchable semantic structure for contextual retrieval.
- **App Intents**: Standardized API surface allowing third-party apps to register actions accessible to the system intelligence layer.
- **Orchestration layer**: Intelligent routing that decides whether a query can be handled on-device, requires Private Cloud Compute, or needs ChatGPT/Gemini fallback.
- **Hybrid architecture**: On-device for speed and privacy (3B model); cloud for complexity; external models for breadth.
- **Hardware gating**: Requires A17 Pro or M-series chips for on-device model execution.

## 3. Key Evidence
- Semantic index enables personal context understanding without centralized data aggregation.
- App Intents toolbox provides structured action interfaces for third-party developers.
- Orchestration layer manages the hybrid execution pipeline.
