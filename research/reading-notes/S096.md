## 1. Core Claim
- Long-term memory is a prerequisite for AI assistants to work effectively, not a feature; Supermemory solves this via a five-layer context stack with structured forgetting, memory graphs, and sub-300ms retrieval.

## 2. Method / Mechanism
- **Five-layer context stack**: Connectors, extractors, RAG, memory graph, user profiles.
- **Three memory types**:
  - **Episodic**: Session-level interaction history and corrections.
  - **Semantic**: Domain knowledge, definitions, concept relationships.
  - **Procedural**: User workflows, code style preferences, explanation formats.
- **Structured forgetting**:
  - Decay policies down-weight old memories by recency and access frequency.
  - Compression collapses redundant entries (claimed >95% token reduction).
  - Triage separates permanent knowledge from session noise before storage.
- **Memory graphs**: Ontology-aware edges track relationship types (not just proximity), enabling contradiction propagation and temporal reasoning.

## 3. Key Evidence
- **92.3% single-session recall**.
- **76.7% multi-session recall**.
- **85.4% LongMemEval-S**.
- **Sub-300ms retrieval**.
- **95% token compression** via structured forgetting.

## 4. Limitations & Gaps
- All metrics are self-reported without independent verification.
- Multi-session recall (76.7%) is notably lower than single-session (92.3%), suggesting session-boundary degradation.
- No discussion of cost per query, data privacy, or compliance.
- Blog post is promotional; no ablation studies or comparison against baselines.
