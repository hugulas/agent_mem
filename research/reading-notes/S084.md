## 1. Core Claim
- Holographic is a lightweight, local, algebraic memory layer optimized for speed and minimal dependencies, contrasting with Hindsight's richer but heavier structured-fact approach.

## 2. Method / Mechanism
- **HRR-style representation**: Memory is stored in a compressed representational space; retrieval is algebraic (bind/unbind/bundle operations) rather than standard chunk-similarity search.
- **Trust scoring**: Memories confirmed across sessions gain weight; contradicted memories lose weight, pushing the store toward self-correction.
- **Local-first SQLite**: Zero external services; attractive for air-gapped, dependency-light, single-user workflows.
- **Minimal tool surface**: Very small API footprint compared to structured-memory alternatives.

## 3. Key Evidence
- Extremely fast local retrieval (sub-millisecond per the article).
- Zero pip dependencies and no external API calls.
- Positioned as the anti-thesis to cloud-heavy, graph-based memory systems.

## 4. Limitations & Gaps
- No published benchmark scores (LongMemEval, LoCoMo, etc.).
- Article is from Vectorize.io (Hindsight's company), so the comparison may have competitive bias.
- Algebraic retrieval may struggle with complex semantic queries that benefit from dense embedding similarity.
- Not clear how HRR handles out-of-vocabulary or multilingual content.

## 5. Relevance to Our Review
- **External memory providers**: Holographic as a minimalist provider option.
- **Academic memory architectures**: HRR as a distinct representational substrate (vector-symbolic architecture).
