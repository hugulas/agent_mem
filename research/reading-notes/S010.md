## 1. Core Claim
Codex CLI implements a **two-layer memory model**: AGENTS.md as the static, user-maintained instruction layer (cross-tool convention under Linux Foundation's Agentic AI Foundation), and Memories as the generated layer where Codex autonomously summarizes prior sessions after a 6-hour idle period.

## 2. Method / Mechanism
- **Layer 1 — AGENTS.md**: Cross-tool instruction file convention (Codex, Cursor, Aider, Jules). Layered discovery: global ~/.codex/AGENTS.md → project walk (root to cwd) → AGENTS.override.md precedence. Default 32 KiB cap (~8,000 tokens). Supports fallback filenames (CLAUDE.md, .cursorrules).
- **Layer 2 — Memories**: Async consolidation after 6h idle. Two-model pipeline: extract model decides what's worth remembering; merge model integrates with existing store. 256 rollout cap, 30-day age-out for unused rollouts and unrecalled memories. Secret redaction before disk write.
- **Recall mechanism**: Reads memory_summary.md whole at session start, token-truncates to fit budget, then uses grep over MEMORY.md for detail — not vector search.
- **Config knobs**: memories.generate_memories, memories.use_memories, memories.extract_model, memories.consolidation_model.
- **Geographic limits**: Not available in EEA, UK, Switzerland at launch.

## 3. Key Evidence
- 32 KiB AGENTS.md ceiling (~8,000 tokens).
- 6-hour idle threshold before consolidation eligibility.
- 256 rollout cap for consolidation.
- 30-day age-out for unused memories.
- Two independent on/off switches for write and read.
- Geographic exclusion: EEA/UK/CH.

## 4. Limitations & Gaps
- **No cross-machine sync**: ~/.codex/memories/ is local only; new machines start from zero.
- **No team sharing**: Memories are per-user; teammates cannot pool generated memories.
- **Generated state not editable**: Hand-editing memories is unsupported; AGENTS.md is the user-editable path.
- **Silent truncation**: 32 KiB cap on AGENTS.md is silent — monorepos with nested instruction files can hit it unnoticed.
- **Cloud Codex underspecified**: Memory persistence across cloud sessions announced but not documented to CLI level.
- **Geographic limits**: EEA/UK/CH users get AGENTS.md only.
- **Recall is grep-based, not semantic**: Cannot find facts with different phrasing from the query.

## 7. Personal Annotation
- The two-layer split (static AGENTS.md + generated Memories) is elegant but creates a tension: users can't easily edit generated memories, and the agent can't edit AGENTS.md.
- Grep-based recall is a surprising choice for 2026 — it suggests OpenAI prioritized predictability and speed over semantic flexibility.
- The 6-hour idle threshold is a pragmatic batching strategy but means memories from a morning session aren't available in an afternoon session on the same day.
- Geographic limits are a significant gap for European users and a compliance red flag.
