## 1. Core Claim
- Karpathy's LLM knowledge base workflow represents a three-stage "ingest/compile/lint" pipeline that bypasses RAG entirely by making markdown the single source of truth, enabling a self-healing, compounding knowledge system.

## 2. Method / Mechanism
- **Three-stage architecture**:
  1. **Ingest**: Raw materials (papers, notes, data) are collected into `raw/`.
  2. **Compile**: LLM processes raw inputs into curated `wiki/` markdown articles.
  3. **Lint**: Automated checks validate consistency, completeness, and structure; issues are filed back into the wiki for correction.
- **Markdown as source of truth**: Unlike RAG, which treats documents as retrievable chunks, this approach transforms inputs into a purpose-built knowledge representation.
- **Self-healing**: Outputs of the linting stage feed back into the wiki, creating a closed improvement loop.
- **Future direction**: Synthetic data generation + fine-tuning as the next phase.
