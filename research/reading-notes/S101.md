## 1. Core Claim
- Memory is the defining capability that transforms stateless LLMs into genuinely adaptive agents; this survey formalizes the design space (2022–early 2026) and identifies critical open challenges in continual consolidation, causally grounded retrieval, learned forgetting, and multimodal embodied memory.

## 2. Method / Mechanism
- **Formalization**: Agent memory as a write–manage–read loop within a POMDP-style agent cycle.
- **3D taxonomy**:
  1. **Temporal scope**: Working / episodic / semantic / procedural memory.
  2. **Representational substrate**: Context-resident text, vector-indexed stores, structured stores (SQL/KG), executable repositories, hybrid stores.
  3. **Control policy**: Heuristic, prompted self-control, learned control (RL).
- **Five mechanism families**:
  1. Context-resident compression
  2. Retrieval-augmented stores
  3. Reflective self-improvement
  4. Hierarchical virtual context
  5. Policy-learned management

## 3. Key Evidence
- **Ablations cited**:
  - Generative Agents without reflection → behavior degenerates within 48 simulated hours.
  - Voyager without skill library → 15.3× slower tech-tree progression.
  - MemoryArena long-context-only baseline → task completion drops from ~80% to ~45% on multi-session tasks.
- **Representative systems table**: RAG (2020), RETRO (2022), ReAct (2022), Reflexion (2023), Generative Agents (2023), Voyager (2023), MemGPT (2024), Agentic Memory (2026), etc.
- Four recent benchmarks analyzed (LoCoMo, MemBench, MemoryAgentBench, MemoryArena).

## 4. Limitations & Gaps
- Survey paper — no new experiments or primary data.

- Open challenges are identified but concrete research agendas are thin.
