## 1. Core Claim
- Graph-based structures are uniquely powerful for agent memory because they naturally model relational dependencies, organize hierarchical information, and support efficient retrieval.
- A systematic life-cycle view (extraction → storage → retrieval → evolution) is necessary to build self-evolving agent memory systems.

## 2. Method / Mechanism
- **Taxonomy**: Classifies agent memory along short-term vs. long-term, knowledge vs. experience, non-structural vs. structural, with a focus on graph-based implementation.
- **Life-cycle framework**: 
  - *Extraction*: Transforming raw interaction data into memory contents.
  - *Storage*: Organizing data efficiently in graph structures (KG, temporal KG, hypergraph).
  - *Retrieval*: Fetching relevant contents to support reasoning.
  - *Evolution*: Updating memory contents over time.
- Surveys open-source libraries and benchmarks supporting graph memory development.

## 3. Key Evidence
- Comprehensive survey coverage of graph-based memory systems including SGMem, Memoria, Nemori, MemoryBank, and Zep.
- Identifies relational dependency modeling, hierarchical organization, and efficient retrieval as the three core strengths of graph memory.
- Collects related resources (papers, data, projects) in a community GitHub repository.

## 4. Limitations & Gaps
- Scope is intentionally limited to graph-based approaches; flat/vector-only systems are underrepresented.
- Does not quantitatively compare graph vs. non-graph approaches on unified benchmarks.
- Future challenges identified (scalability, dynamic evolution, multi-agent graph merging) remain open research directions.

## 5. Relevance to Our Review
- **Knowledge graph memory**: The authoritative survey source for our KG memory subsection.
- **Academic memory architectures**: Provides the theoretical grounding for why graphs are favored in advanced agent memory research.
