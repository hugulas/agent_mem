## 1. Core Claim
- Supermemory is a state-of-the-art long-term and short-term memory and context infrastructure for AI agents, providing perfect recall about users to build more intelligent, personalized, and consistent agents.

## 2. Method / Mechanism
- **5-layer context stack**: Connectors, extractors, RAG, memory graph, user profiles.
- **Ingestion**: Accepts text, conversations, files (PDF, images, docs), and videos.
- **Three context delivery modes**:
  - **Memory API**: Learned, evolving user facts (handles updates, temporal changes, forgetfulness).
  - **User Profile**: Static + dynamic facts the agent should always know.
  - **RAG**: Advanced semantic search with metadata filtering and contextual chunking.
- All three modes share the same context pool when using the same user ID (`containerTag`).

## 3. Key Evidence
- Claims state-of-the-art performance on LongMemEval and LoCoMo.
- Supports any data type, including video.
- Managed RAG platform with contextual chunking.

## 4. Limitations & Gaps
- Cloud API only — no self-host option (proprietary).
- "Perfect recall" is marketing hyperbole, not an empirical claim.
- Benchmark details and methodology not shown on this overview page.
- No discussion of latency, cost, or privacy governance.

## 5. Relevance to Our Review
- **External memory providers**: Supermemory as a closed-source, cloud-native memory platform.
