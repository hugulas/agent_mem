## 1. Core Claim
Claude Code manages context pressure through a **multi-tier compaction pipeline** with distinct layers: budget reduction for individual tool outputs, snip compact for temporal depth, microcompact for cache overhead via the cache_edits API, context collapse for extreme history lengths, and autocompact as a reactive last resort.

## 2. Method / Mechanism
- **Tool result budgets**: Individual tool outputs capped at configurable sizes (50K/200K characters) to prevent a single verbose output from consuming disproportionate context.
- **Snip compact**: Trims temporal depth — removes oldest messages first when context grows too long.
- **Microcompact**: Uses the `cache_edits` API to reduce cache overhead from repeated file edits.
- **Context collapse**: Manages very long histories by aggressively summarizing older turns.
- **Autocompact**: Reactive semantic compression triggered when all other layers fail to bring context under budget.
- **55K token tool definitions**: Full tool schemas consume significant context even before user messages.
- **Prompt Cache**: Anthropic's prompt caching reduces cost for repeated context segments.

## 3. Key Evidence
- Tool result budgets: 50K/200K character caps.
- 55K token tool definitions count toward context before any user input.
- Five distinct compaction layers each operating at different cost-benefit trade-offs.
- Per-turn pipeline execution means compaction runs before every model call.
