> Abstract
> As large language models engage in extended reasoning tasks, they accumulate signifi-
> cant state – architectural mappings, trade-off decisions, codebase conventions – within the
> context window. This understanding is lost when sessions reach context limits and undergo
> lossy compaction.
> We propose contextual memory virtualisation (CMV), a system that
> treats accumulated LLM understanding as version-controlled state. Borrowing from operat-
> ing system virtual memory, CMV models session history as a Directed Acyclic Graph (DAG)
> with formally defined snapshot, branch, and trim primitives that enable context reuse across
> independent parallel sessions. We introduce a three-pass structurally lossless trimming al-
> gorithm that preserves every user message and assistant response verbatim while reducing
> token counts by a mean of 20% and up to 86% for sessions with significant overhead by strip-
> ping mechanical bloat such as raw tool outputs, base64 images, and metadata. A single-user
> case-study evaluation across 76 real-world coding sessions demonstrates that trimming re-
> mains economically viable under prompt caching, with the strongest gains in mixed tool-use
> sessions, which average 39% reduction and reach break-even within 10 turns. A reference
> implementation is available at https://github.com/CosmoNaught/claude-code-cmv.

> Introduction
> Extended work sessions with LLM coding agents build cumulative state within the context
> window. Architecture gets mapped, trade-offs get weighed, decisions are recorded, conventions
> are learned.
> After 30 minutes of deep work, the model holds a mental model of an entire
> codebase, built at significant computational cost in both time and tokens. When the context
> window fills, native compaction (e.g., /compact in Claude Code [Anthropic, 2024b]) summarises
> this state into a few sentences.
> In observed sessions, autocompaction reduced 132k tokens
> of accumulated message state to 2.3k—a 98% reduction (Figure 1)—discarding the nuanced
> understanding that took an entire session to build. Each new session starts from scratch.
> This is a fundamental inefficiency.
> The cost of building context is paid repeatedly, and
> the resulting understanding is never preserved in a reusable form. Existing approaches address
> fragments of this problem. Retrieval-Augmented Generation (RAG) [Lewis et al., 2020] aug-
> ments prompts with retrieved documents but does not preserve conversational state. MemGPT
> [Packer et al., 2023] applies OS-inspired paging to swap context segments in and out of the
> window, but is limited to a single session and relies on the model to manage its own memory.
> Memory plugins persist summary facts across sessions but lose the full conversational nuance.
> Native session utilities (/rewind, --fork) provide within-session undo and one-off copies but
> lack named states, lineage tracking, or context cleanup.

## 1. Core Claim
- Native compaction (e.g., Claude Code /compact) is structurally lossy: it reduced 132k tokens to 2.3k (98% reduction) in observed sessions, discarding nuanced understanding built at significant computational cost.
- Contextual Memory Virtualisation (CMV) treats accumulated LLM understanding as version-controlled state, enabling structurally lossless trimming and context reuse across independent parallel sessions.

## 2. Method / Mechanism
- **DAG-based state management**: Models session history as a Directed Acyclic Graph with formally defined snapshot, branch, and trim primitives.
- **Three-pass structurally lossless trimming algorithm**: Preserves every user message and assistant response verbatim while stripping mechanical bloat (raw tool outputs, base64 images, metadata).
- **Context reuse across sessions**: Named states and lineage tracking enable fork/merge semantics for parallel sessions.
- **Prompt caching awareness**: Trimming is designed to remain economically viable under cached prompt pricing.

## 3. Key Evidence
- Single-user case study across **76 real-world coding sessions**.
- Mean token reduction of **20%**, up to **86%** for sessions with significant overhead.
- Mixed tool-use sessions average **39% reduction** and reach break-even within **10 turns** under prompt caching.
- All sessions combined: mean 82k tokens → 76 sessions, 20% reduction, 12% overhead, break-even at 35 turns.

## 4. Limitations & Gaps
- Single-user case study limits generalizability to multi-user or collaborative scenarios.
- Focused on coding agents; generalization to non-coding conversational or task-oriented agents is unverified.
- Structurally lossless trimming preserves messages but does not create semantic summaries for cross-session transfer.
- The DAG model adds client-side complexity not present in standard chat interfaces.

## 5. Relevance to Our Review
- **Context compression**: Offers a principled alternative to lossy LLM summarization by trimming mechanical bloat instead.
- **Academic memory architectures**: The OS virtual-memory analogy (snapshot, branch, trim) provides a novel conceptual framework.

## 7. Personal Annotation
- The DAG-based session model is elegant and maps well onto software engineering workflows where branching and merging are natural.
- The distinction between "structurally lossless" (trimming bloat) and "semantically lossless" is important; CMV does the former but not the latter.
- Could be integrated with Slipstream (async validation) to handle cases where trimming alone is insufficient.
