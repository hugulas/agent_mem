## 1. Core Claim
OpenClaw's builtin memory engine is a **local-first, SQLite-backed hybrid search system** that combines FTS5 BM25 keyword search with vector semantic search, using Markdown files as the canonical source of truth and a derived SQLite index for fast retrieval.

## 2. Method / Mechanism
- **Hybrid search (union, not intersection)**: Vector search (70% weight) + BM25 keyword search (30% weight). Results from either search contribute; missing one score defaults to 0 rather than filtering out.
- **FTS5 with CJK trigram tokenization**: Enables full-text search across languages including Chinese, Japanese, Korean.
- **Vector search via embeddings**: sqlite-vec acceleration for efficient similarity search over chunked embeddings.
- **7 embedding providers**: OpenAI, Gemini, Voyage, Mistral, DeepInfra, Ollama, and Local GGUF (~0.6GB).
- **File-as-canonical architecture**: Markdown files are the source of truth; SQLite index is derived and can be rebuilt.

## 3. Key Evidence
- 7 embedding providers supported with auto-fallback (local first, then cloud).
- Hybrid search weights: 70% vector / 30% BM25.
- CJK trigram tokenization specifically mentioned for East Asian language support.
- sqlite-vec for accelerated vector operations.
