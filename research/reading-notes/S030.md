## 1. Core Claim
OpenClaw's memorySearch subsystem exposes fine-grained tunable parameters—candidate multiplier 4, MMR lambda 0.7, 30-day temporal decay half-life, 50,000-entry cache, and 1,500 ms watch debounce—that significantly affect recall quality and latency.

## 2. Method / Mechanism
- Candidate multiplier (4×): expands the initial retrieval pool before re-ranking
- MMR (Maximal Marginal Relevance) lambda (0.7): balances relevance vs. diversity in returned results
- Temporal decay: half-life of 30 days, biasing toward recent memories
- Cache layer: maxEntries 50,000 for embedding lookup acceleration
- File watcher debounce: 1,500 ms to batch rapid filesystem changes before re-indexing

## 3. Key Evidence
- Candidate multiplier: 4
- MMR lambda: 0.7

- Cache maxEntries: 50,000
- watchDebounce: 1,500 ms
