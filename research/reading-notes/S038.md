## 1. Core Claim
Hermes Agent uses a predictable local filesystem structure for state and memory: `~/.hermes/config.yaml` for configuration, `state.db` for runtime state, `skills/` for capability modules, and `memories/MEMORY.md` plus `USER.md` for persistent agent memory.

## 2. Method / Mechanism
- Filesystem-based state management in `~/.hermes/` directory
- `config.yaml`: central configuration file
- `state.db`: SQLite database for runtime state
- `skills/`: directory containing skill modules
- `memories/MEMORY.md`: agent's persistent memory document
- `USER.md`: user-specific context document
- Built-in MEMORY.md/USER.md are always active regardless of external provider selection (per S013)

## 3. Key Evidence
- Directory: `~/.hermes/`
- Config: `config.yaml`
- State: `state.db`
- Skills: `skills/`
- Memory: `memories/MEMORY.md`
- User context: `USER.md`
