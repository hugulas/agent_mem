## 1. Core Claim
- VS Code Copilot's Memory tool (preview) provides three scoped persistence layers—User, Repository, and Session—with User memory persisting across workspaces and the first 200 lines auto-loaded into context, all stored locally on the developer's machine.

## 2. Method / Mechanism
- **Three scopes**:
  - **User memory**: Persists across all workspaces and projects; global developer preferences and patterns.
  - **Repository memory**: Project-specific context tied to the current codebase.
  - **Session memory**: Ephemeral context for the current interaction only.
- **Auto-loading**: First 200 lines of relevant memory are automatically injected into the prompt context.
- **Local storage**: All memory data is stored locally, addressing privacy concerns for source code.

## 3. Key Evidence
- **200 lines auto-loaded**: Default context injection from memory into active sessions.
- **Local-only storage**: No cloud upload of memory data.
- **Preview status**: Feature is in preview, subject to change.
