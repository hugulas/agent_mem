## 1. Core Claim
- The context window of an LLM is not memory—it is L1 cache. The field currently treats it as the entire memory system, with no L2, virtual memory, or paging.
- This architectural omission causes measurable waste: 21.8% of tokens in production sessions are structural waste. Demand paging (Pichay) can reduce context consumption by up to 93%.

## 2. Method / Mechanism
- **Pichay**: Transparent proxy between client and inference API that interposes on the message stream.
- **Eviction**: Removes stale content (unused tool schemas, duplicated content, stale tool results) from context.
- **Page fault detection**: Detects when the model re-requests evicted material and faults it back in.
- **Working-set pinning**: Identifies and pins frequently accessed pages based on fault history.
- **Memory hierarchy**: Proposes L1 (context window) → L2 (fault-driven cache) → L3 (compaction) → persistent storage.

## 3. Key Evidence
- **Production instrumentation**: 857 sessions, 54,170 API calls, 4.45 billion effective input tokens analyzed.
- **Structural waste breakdown**: 21.8% total waste—11.0% unused tool schemas, 2.2% duplicated content, 8.7% stale tool results (reprocessed at median 84.4× amplification).
- **Offline replay**: 1.4 million simulated evictions with 0.0254% fault rate.
- **Live deployment**: 681 turns, context reduction up to 93% (5,038KB → 339KB).

## 4. Limitations & Gaps
- Under extreme sustained pressure, the system exhibits thrashing (repeated fault-in of evicted content), as expected from OS theory.
- Cross-session memory remains the remaining frontier; Pichay only manages within-session context.
- Live deployment scale (681 turns) is modest compared to the offline analysis (4.45B tokens).
- Requires proxy interposition, which may not be feasible for all API consumers or local inference setups.

## 5. Relevance to Our Review
- **Context compression**: The strongest OS-inspired analogy in the surveyed literature.
- **Academic memory architectures**: Proposes a full memory hierarchy (L1–L3 + persistent) for LLM systems.
