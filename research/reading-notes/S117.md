## 1. Core Claim
- Existing KV cache compression methods effective for general LLMs suffer suboptimal performance in GUI agents because GUI attention patterns exhibit uniform high-sparsity across all transformer layers, unlike general visual tasks where sparsity varies by layer.
- A training-free, GUI-specific KV cache compression framework (ST-Lite) with dual-branch spatial and trajectory-aware scoring achieves 2.45× decoding acceleration at 10–20% cache budget without performance degradation.

## 2. Method / Mechanism
- **ST-Lite framework**: Training-free KV cache compression tailored for GUI agents.
- **Component-centric Spatial Saliency (CSS)**: Preserves structural integrity of interactive UI elements by evaluating local neighborhood saliency.
- **Trajectory-aware Semantic Gating (TSG)**: Mitigates historical redundancy by dynamically filtering visually repetitive KV pairs within the interaction trajectory.
- **Dual-branch scoring policy**: Combines CSS and TSG for comprehensive cache eviction decisions.

## 3. Key Evidence
- With only **10–20% cache budget**, ST-Lite achieves **2.45× decoding acceleration**.
- Maintains comparable or superior performance vs. full-cache baselines on GUI agent benchmarks.
- Outperforms general-purpose compression methods (e.g., SnapKV) that are misaligned with GUI attention patterns.
- Ablation shows both CSS-only and TSG-only branches contribute, with full ST-Lite achieving the best balance.

## 4. Limitations & Gaps
- Training-free approach may be suboptimal compared to learned compression methods if training data is available.
- Evaluated on specific GUI agent benchmarks; generalization to other VLM tasks (e.g., video understanding, robotics) is unverified.
- Assumes high-resolution screenshots are the primary input; text-heavy GUI tasks may have different redundancy patterns.
- Does not address cross-session memory—only within-session KV cache compression.

## 5. Relevance to Our Review
- **GUI agents memory**: A specialized but important domain where memory constraints are particularly severe due to visual inputs.
- **Context compression**: Demonstrates that domain-aware compression outperforms generic methods.
