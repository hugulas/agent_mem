## 1. Core Claim
- Agent-native memory architecture inverts the traditional pipeline: the same LLM that reasons also curates, structures, and retrieves knowledge, eliminating semantic drift caused by external-service memory pipelines.

## 2. Method / Mechanism
- **Hierarchical Context Tree**: Domain → Topic → Subtopic → Entry, stored as human-readable Markdown files on the local filesystem.
- **Adaptive Knowledge Lifecycle (AKL)**: Each entry carries importance scoring, maturity tiers, and recency decay.
- **5-tier progressive retrieval**: Resolves most queries at sub-100 ms without LLM calls; escalates to agentic reasoning only for novel questions.
- **Zero external infrastructure**: No vector database, graph database, or embedding service.

## 3. Key Evidence
- **LoCoMo**: State-of-the-art accuracy (96.1% per S082 reading-log entry).
- **LongMemEval-S**: Competitive score of 92.8%.
- **Latency**: Cold query latency p50 ~1.2s, p95 ~1.6s, p99 ~2.3s (from numeric claims table).
- All knowledge stored as local Markdown files; no external dependencies.

## 4. Limitations & Gaps
- Only evaluated on two benchmarks (LoCoMo and LongMemEval); generalization to other domains unproven.
- Local filesystem storage may not scale to massive corpora or multi-agent shared environments.
- Single-team/authors (ByteRover) — potential benchmarking bias.
- No discussion of multi-modal memory or privacy governance beyond local storage.
