> ABSTRACT
> Large language models face challenges in long-context question answering, where
> key evidence of a query may be dispersed across millions of tokens. Existing
> works equip large language models with a memory buffer that is dynamically up-
> dated via a linear document scan, also known as the “memorize while reading”
> methods. While this approach scales efficiently, it suffers from pruning of latent
> evidence, information loss through overwriting, and sparse reinforcement learn-
> ing signals. To tackle these challenges, we present ReMemR1, which integrates
> the mechanism of memory retrieval into the memory update process, enabling
> the agent to selectively callback historical memories for non-linear reasoning. To
> further strengthen training, we propose a multi-level reward design, which com-
> bines final-answer rewards with dense, step-level signals that guide effective mem-
> ory use. Together, these contributions mitigate information degradation, improve
> supervision, and support complex multi-hop reasoning. Extensive experiments
> demonstrate that ReMemR1 significantly outperforms state-of-the-art baselines on
> long-context question answering while incurring negligible computational over-
> head, validating its ability to trade marginal cost for robust long-context reasoning.
> Our code is available at https://github.com/syr-cn/ReMemR1.

> INTRODUCTION
> Reasoning over vast, multi-document contexts remains a critical bottleneck for large language mod-
> els (LLMs) (Hsieh et al., 2024; Team et al., 2024; Beltagy et al., 2020; Ding et al., 2023; Child et al.,
> 2019). This capability is crucial for real-world applications, such as synthesizing legal precedents
> or reviewing scientific literature, where critical evidence for a single query can be scattered across
> millions of tokens.
> However, the quadratic complexity of attention mechanisms makes it difficult
> for LLMs to track long-range dependencies and faithfully synthesize disparate information into a
> coherent answer.
> To mitigate this, two primary paradigms have emerged. The first is Full-Text Context Retrieval
> (Figure 1(a)), where a retriever fetches relevant chunks from a corpus to form a prompt (Jin et al.,
> 2025; Song et al., 2025; Shi et al., 2025; Du et al., 2026; 2025a). While widely used, this approach
> presents the LLM with fragmented, partial information and suffers from a heavy storage burden for
> the vector index. Alternatively, recent research explores the “memorize while reading” paradigm
> (Yu et al., 2025a; Li et al., 2025; Wang et al., 2025b; Shi et al., 2026) to handle infinite contexts
> linearly. As shown in Figure 1(b), this framework employs a memory agent that digests documents
> sequentially. At each step, the agent consumes a document chunk ct together with its previous
> memory mt and compresses them into a new memory mt+1. After a single linear pass through the
> entire document, the agent uses this final memory mT buffer to generate an answer for the given
> question. This reduces the complexity of long-context question answering to linear time.

## 1. Core Claim
- "Memorize while reading" methods (linear memory buffer updates) suffer from three critical flaws: pruning of latent evidence, information loss through overwriting, and sparse reinforcement learning signals.
- Integrating memory retrieval into the memory update process—enabling selective callbacks to historical memories—mitigates information degradation and supports complex multi-hop reasoning with negligible computational overhead.

## 2. Method / Mechanism
- **ReMemR1**: Integrates retrieval into the memory update process, enabling non-linear memory visiting.
- **Selective callback**: Agent can revisit and reason over historical memories rather than only the current buffer state.
- **Multi-level reward design**: Combines final-answer rewards with dense, step-level signals that guide effective memory use.
- **Linear-time scaling**: Maintains efficient scaling despite the added retrieval mechanism.

## 3. Key Evidence
- Significantly outperforms state-of-the-art baselines on long-context question answering benchmarks (HotpotQA, Natural Questions, etc.).
- Tested with up to **6,400 context documents**, demonstrating robustness at extreme scale.
- Incurs **negligible computational overhead** despite the callback mechanism.
- Maintains strong performance as context length increases, unlike linear-update baselines that degrade.

## 4. Limitations & Gaps
- Focused on question-answering over static documents rather than interactive, multi-turn agents.
- The callback mechanism adds complexity to the memory update loop that may be challenging to implement in real-time conversational systems.
- Multi-level reward design requires careful tuning; dense rewards may inadvertently incentivize superficial memory use.
- No comparison against graph-based or hierarchical memory systems that also enable non-linear access.

## 5. Relevance to Our Review
- **Academic memory architectures**: Represents a novel direction in memory update mechanisms by integrating retrieval into writing.
- **RL-based memory**: The multi-level reward design advances RL training for memory-augmented systems.
